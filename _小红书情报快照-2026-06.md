## §4 新概念追踪：Loop Engineering（已核实，已并入 KB）

- **是什么**：harness 装备的是「单次运行」，**Loop Engineering 是其上一层**——那个不停**定时唤醒 / 派生子 agent / 自我投喂**、驱动 agent 一遍遍跑的外层循环。
- **出处（已联网核实）**：起于 Peter Steinberger 在 X 的帖子（2026-06-07），二周内重塑社区话语；综述见 Cobus Greyling《Loop Engineering》、explainx 等。它**不取代 harness**，是 `prompt → context → harness → loop` 的第四跳。
- **小红书印证**：@雾霁《Harness Engineering 已经过时了。Loop Eng…》热评精炼——「Harness 定义 Agent 跑多稳的**底线**，Loop 决定能力长多高的**上限**；一个在外部给规则，一个在内部长智慧」。
- **已并入 KB**：[[_事实基线-2026-06]] §2 新增行、[[OVERVIEW-发展总脉络]] 2026-06 条与主时间线、[[02]] §3 时间线 + §4.7 末「从 harness 到 loop」延伸 + 参考文献。
- **顺带补**：同月口径还把 **OpenAI《Harness engineering》(Codex, 2026-03-04)** 补进基线与 [[02]]，坐实"harness engineering = 行业通用词，非单家说法"。
