§1 一句话结论 #

本库押注的「Harness 是独立工程层」这件事，已经从英文圈一手博客（Anthropic / OpenAI / Cursor）外溢成中文社区的当红词——「Harness Engineering」在小红书出现单条收藏 7000+、多条 1000+ 的科普视频，DeepSeek 已挂出「Harness 工程师」岗、字节出了「Harness 手册」。与此同时，社区已经在追下一跳「Loop Engineering」（2026-06，本快照触发的最有价值发现，已核实并并入 KB）。但社媒内容质量参差、且有硬错（最典型：把 Harness Engineering 说成"OpenAI 2026 年 2 月首提"），所以本轮策略是：只把已联网核实的新进展写入正文，把社媒信号与面经留在本快照、清楚标注未核验。

§4 新概念追踪：Loop Engineering（已核实，已并入 KB） #

  • 是什么：harness 装备的是「单次运行」，Loop Engineering 是其上一层——那个不停定时唤醒 / 派生子 agent / 自我投喂、驱动 agent 一遍遍跑的外层循环。
  • 出处（已联网核实）：起于 Peter Steinberger 在 X 的帖子（2026-06-07），二周内重塑社区话语；综述见 Cobus Greyling《Loop Engineering》、explainx 等。它不取代 harness，是 prompt → context → harness → loop 的第四跳。
  • 小红书印证：@雾霁《Harness Engineering 已经过时了。Loop Eng…》热评精炼——「Harness 定义 Agent 跑多稳的底线，Loop 决定能力长多高的上限；一个在外部给规则，一个在内部长智慧」。
  • 已并入 KB：事实基线-2026-06 §2 新增行、发展总脉络 2026-06 条与主时间线、02 §3 时间线 + §4.7 末「从 harness 到 loop」延伸 + 参考文献。
  • 顺带补：同月口径还把 OpenAI《Harness engineering》(Codex, 2026-03-04) 补进基线与 02，坐实"harness engineering = 行业通用词，非单家说法"。
